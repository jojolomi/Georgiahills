# Mobile Performance Roadmap (Quality-First)

This roadmap is optimized for mobile PageSpeed outcomes and user-perceived speed.

## Targets

The canonical target model is tracked in:
- scripts/audit/perf/mobile-template-targets.json

Validation is enforced by:
- scripts/audit/perf/check-mobile-template-targets.js
- npm run perf:mobile-targets
- npm run preflight:pr

## Step Status

1. Measurement and target model
- Status: Completed (enforced)
- Output: Template families, priorities, and soft mobile budgets are locked in JSON and validated in CI.

2. Critical-path audit by template
- Status: Completed (enforced for priority templates)
- Output now: Automated report at scripts/audit/perf/template-critical-path-report.json generated by npm run perf:critical-path:audit.
- Enforcement now: npm run perf:critical-path:budget fails priority templates on blocking CSS, non-deferred external scripts, missing required hero preload, and inline-script budget breaches.

3. LCP path optimization
- Status: Completed for priority templates
- Output now: Priority-template head optimizations applied (destination hero preload; non-blocking stylesheet loading on key templates; destination Firebase bootstrap deferred).
- Validation now: Priority templates pass critical-path budget guardrails and lighthouse config thresholds.

4. Render-blocking and CSS strategy
- Status: Completed for priority templates
- Output now: Non-blocking stylesheet loading pattern applied on priority templates.
- Enforcement now: blocking stylesheet count is audited and CI-failed by critical-path budget guard.

5. Route-level JS separation and runtime slimming
- Status: In progress
- Output now: Article runtime split is enforced in CI (`npm run perf:article-runtime-split`) and non-critical loaders are deferred via idle scheduling on non-conversion routes.
- Output needed: Continue long-tail route-specific runtime extraction beyond current split.

6. Third-party and late-hydration governance
- Status: Completed (enforced for configured templates)
- Output now: Guard script enforces non-blocking third-party scripts and non-blocking external stylesheet pattern across all configured templates (`npm run perf:third-party-governance`).
- Output now: Vendor policy file codified at `scripts/audit/perf/third-party-governance-policy.json`.

7. HTML and DOM simplification
- Status: In progress
- Output now: Automated DOM-size audit and report are enforced in preflight (`npm run perf:dom-size:audit`).
- Output needed: Reduce inline style/handler density and trim non-critical nodes on priority templates.

8. Network/cache/TTFB governance
- Status: Completed (enforced in deploy/release workflows)
- Output now: Netlify cache policy headers and CI guard added (`npm run ops:netlify-cache-policy`) for key assets and html revalidation.
- Output now: Delivery SLO script added with redirect-chain, TTFB, and compression checks in warn/strict modes (`npm run ops:delivery:slo:warn`, `npm run ops:delivery:slo:strict`).
- Enforcement now: strict checks wired into staging/production post-deploy workflows; warn-mode checks wired into local release verification.
- Next tightening path: refine SLO thresholds from production telemetry windows and move selected warn checks to strict.

9. Verification loop and CI budgets
- Status: Completed for current baseline
- Output now: Mobile target model + critical-path audit + critical-path budget + governance/cache checks are wired into `preflight:pr`.
- Output now: Lighthouse config guardrails are enforced with stronger thresholds.
- Next tightening path: reduce numeric budgets iteratively after additional production telemetry cycles.

10. Architecture cleanup last
- Status: Pending
- Output needed: Migration cleanup once score/stability goals are met.

## Execution Notes

- Prioritize template family order: home -> booking -> destination -> blog -> article.
- Tighten budgets only after two consecutive stable runs.
- Do not merge route-level refactors without template-level before/after evidence.
